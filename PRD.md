# Yonsei Dawoong Hospital Website Proposal

## 1. Overview

This project is a school demo for a multi-page website for Yonsei Dawoong Hospital. The site is designed for patients, with a strong focus on the U.S. Army military personnel stationed in Pyeongtaek, Samsung factory workers in the region, and local residents. The primary goal is customer acquisition through clear service information, doctor details, appointment request capabilities, and easy access to location information.

## 2. Objectives

- Present Yonsei Dawoong Hospital as a professional and trustworthy medical provider.
- Help target patients find services quickly, especially U.S. military and local workers.
- Provide clear doctor profiles and service summaries.
- Offer a demo appointment request form without backend booking integration.
- Make hospital location and directions easy to find.

## 3. Target Users

- U.S. Army military personnel near Pyeongtaek.
- Samsung factory workers and employees in the area.
- Local regional patients looking for hospital services.
- The project will be evaluated by the course professor.

## 4. Key Features

- Service information pages describing what the hospital offers.
- Doctor information and credentials.
- Appointment request demo form for patients to contact the hospital.
- Location page with directions and contact details.
- A dedicated section for U.S. military / Pyeongtaek support information.
- A professional, trust-building presentation style.

## 5. Proposed Site Structure

Minimum viable pages:

1. Home
2. Services
3. Doctors
4. Appointment Request
5. Location
6. US Military / Pyeongtaek Support

## 6. User Flow

Primary demo flow:

1. User visits the site and lands on the Home page.
2. User views doctor details and hospital credentials.
3. User reviews service information.
4. User submits a demo appointment request form.
5. User finds location and direction details easily.

## 7. Acceptance Criteria

The demo will be considered complete when:

- Doctor profiles and credentials are visible and readable.
- Service descriptions are clearly presented.
- The appointment request page displays a working demo form.
- The location page provides address and directions.
- U.S. military support information is included.
- Navigation is easy and page content loads quickly.

## 8. Assumptions and Constraints

- The site is a static demo; no backend booking integration is required.
- Content may use placeholder or sample data for doctor details and services.
- The primary focus is on ease of navigation and fast loading, not on mobile polish.

## 9. Non-Functional Requirements

- Easy-to-use navigation across all pages.
- Fast page load performance.
- Professional visual tone.
- Clear structure for first-time visitors.

## 10. Timeline

- Complete the proposal and design plan immediately.
- Build the multi-page static site and add demo content before Friday.
- Finalize the presentation for the professor by the due date.

## 11. Review and Stakeholder

- Primary stakeholder: Doctors and Professionals 
- The demo should show a completed static website with a clear patient-focused flow.
